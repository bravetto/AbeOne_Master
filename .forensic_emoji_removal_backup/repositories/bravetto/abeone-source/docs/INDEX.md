## üìù Documentation Standards

### File Naming
- Use `SCREAMING_SNAKE_CASE.md` for documentation files
- Use descriptive names that indicate content
- Use `README.md` for directory overviews

### Structure
- Start with title and brief description
- Include quick navigation table when applicable
- Use clear sections with headers
- Include code examples where relevant
- Link to related documentation

### Maintenance
- Keep documentation up to date with code changes
- Archive outdated documentation to `archive/`
- Consolidate duplicate documentation
- Update this index when adding new documentation
